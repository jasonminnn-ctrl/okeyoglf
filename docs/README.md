# OkeyGolf OS 문서 기준 구조

이 `docs` 폴더는 OkeyGolf OS 개발본부 프로젝트의 기준 문서 저장소다.

## 폴더 구성

### 1. `standards/`
현재 기준 문서 00~10을 보관하는 폴더다.

이 폴더의 문서를 기준 원본(SSOT)으로 본다.

- 00_프로젝트_개요
- 01_메뉴구조_정본
- 02_업종별_정의
- 03_ROS_엔진정의
- 04_프롬프트정책
- 05_산출물포맷
- 06_개발로드맵
- 07_시장조사_운영정의
- 08_외부연동_정의
- 09_기능잠금_운영정책
- 10_운영형_데이터보드_정의

### 2. `legacy/`
기존 단계 문서, 요약 문서, 임시 기준 문서를 보관하는 폴더다.

이 폴더의 문서는 참고용이며, 현재 기준 판단은 `standards/` 문서를 우선한다.

## 운영 원칙

- 기준 원본(SSOT)은 `docs/standards/` 문서다.
- Google Docs는 공유/검토용 미러로 사용한다.
- 현재 위치, 완료 단계, 다음 우선순위는 `standards/06_개발로드맵.md`에서만 관리한다.
- 외부 연동, 아임웹, OpenAI, 알림/발송 기준은 `standards/08_외부연동_정의.md`에서 관리한다.
- 기능 잠금, enabled/locked/hidden 기준은 `standards/09_기능잠금_운영정책.md`에서 관리한다.
- 운영형 데이터보드, CSV/XLSX, KPI 차트 기준은 `standards/10_운영형_데이터보드_정의.md`에서 관리한다.
- 기존 레거시 문서는 참고용으로만 두며, 기준 판단은 `standards/`를 우선한다.

## 문서 수정 순서

방향 변경이나 정책 변경이 생기면 아래 순서로 반영한다.

1. `06_개발로드맵` 현재 위치/우선순위 수정
2. 관련 세부 기준 문서 수정 (`07`, `08`, `09`, `10` 등)
3. 코드 / DB / UI 반영
4. 필요 시 legacy 문서 정리
