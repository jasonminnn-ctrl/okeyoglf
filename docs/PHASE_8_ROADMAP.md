# OkeyGolf OS — 공식 8단계 외부연동 로드맵

> 작성: 2026-03-11 (8-0 준비정리 시점)

---

## 1. 8단계 범위 (이번 단계)

### 1-1. OpenAI 서버사이드 호출 구조

- Edge Function 기반 OpenAI API 호출
- 프론트엔드에서 API 키 노출 금지 (서버사이드 전용)
- 요청 → Edge Function → OpenAI → 응답 → 클라이언트
- rate limit / 에러 핸들링 / 재시도 구조
- 비용 추적을 위한 usage 로깅 (token count)

### 1-2. 운영자 "외부 연동" 탭

OperatorPage에 신규 탭 추가 (고객 비노출):

| 섹션 | 설명 |
|---|---|
| 연동 상태 대시보드 | 각 연동의 현재 상태 (연결됨/미연결/오류) |
| OpenAI 연동 | API 키 등록, 모델 설정, 사용량 모니터링 |
| 아임웹 회원/주문 연동 | webhook URL, 동기화 상태, 최근 이벤트 |
| 상품 → 멤버십 매핑 | 아임웹 상품 ID ↔ membershipCode 매핑 테이블 |
| 포인트상품 → 크레딧 매핑 | 아임웹 포인트상품 ID ↔ creditAmount 매핑 |
| 알림/발송 연동 | 카카오 알림톡/SMS/이메일 provider 설정 |
| 동기화 이력 / 예외 처리 | 실패 건 목록, 수동 재처리 버튼 |

### 1-3. 아임웹 연동 구조

**우선순위:**

| 순위 | 항목 | 설명 |
|---|---|---|
| 1 | 상품 → membershipCode 매핑 | 아임웹 상품 구매 → 자동 멤버십 부여 |
| 2 | 포인트상품 → creditAmount 매핑 | 아임웹 포인트상품 구매 → 자동 크레딧 충전 |
| 3 | 주문 처리 이력 / 예외 처리 | 결제 실패, 취소, 환불 대응 |
| 4 | 회원 매칭 | 아임웹 회원 ↔ OkeyGolf 조직 연결 |
| 5 | 실제 로그인/인증 연계 | 아임웹 SSO 또는 토큰 기반 인증 (후순위) |

> 원칙: 로그인보다 운영 매핑이 먼저. 운영자가 주문 → 멤버십/크레딧 흐름을 이해하고 관리할 수 있게.

### 1-4. 알림/발송 provider 구조

- provider 인터페이스 정의 (send, getStatus)
- 카카오 알림톡 placeholder
- SMS placeholder
- 이메일 placeholder
- 실제 연결은 provider별 API 키 등록 후 활성화

### 1-5. 예외 처리 / 수동 재처리

- 동기화 실패 상태: `sync_failed`, `retry_pending`, `manually_resolved`
- 운영자가 실패 건을 보고 수동 재처리 가능
- 재처리 이력 기록

## 2. 9단계로 미루는 것

| 항목 | 사유 |
|---|---|
| 운영자 요청 접수함 고도화 | 실제 고객 요청 볼륨 확보 후 |
| 전담 컨설턴트 내부 메모/우선순위/결과 업로드 | 컨설턴트 워크플로우 고도화 |
| 고객별 이력 고도화 | DB 기반 이력 관리 필요 |
| 실제 처리 워크플로우 강화 | 운영 안정화 후 |
| 실제 상품형 서비스 운영 로직 고도화 | 과금/결제 시스템 완성 후 |
| 저장공간 유료 과금/상품 판매 | 결제 인프라 완성 후 |
| 아임웹 SSO/로그인 연계 | 매핑 구조 안정 후 |

## 3. 실제 연결 vs placeholder 구분 원칙

| 구분 | 기준 |
|---|---|
| **실제 연결** | API 키가 등록되고, Edge Function이 실제 호출을 수행하며, 응답을 처리하는 상태 |
| **placeholder** | UI 구조와 데이터 모델은 존재하지만 실제 외부 호출 없음. "연결 준비" / "미연결" 상태로 표시 |

> 원칙: 사용자가 placeholder를 "작동 중"으로 오해하지 않게 명확히 상태 표시.

## 4. 기능 잠금 정책 (8단계 기준)

### 계속 잠금 유지
- 외부 수집 기반 고급 시장조사 (RESEARCH_ADVANCED) — trial/standard
- 전담 컨설턴트 전체 기능 — trial/standard
- 결과 전달 (RESULT_DELIVER) — trial/standard

### 8단계 구현 시 선택적 활성화
- OpenAI 기반 생성 기능 — API 키 등록 완료 시
- 아임웹 연동 기능 — webhook 연결 완료 시

### UI 노출하되 "연결 준비" 상태
- 운영자 외부 연동 탭 → 각 섹션에 연결 상태 표시
- 알림/발송 provider → "미연결" 상태 표시

### 아직 숨김
- 아임웹 SSO/로그인 (9단계)
- 저장공간 유료 과금 (9단계)
