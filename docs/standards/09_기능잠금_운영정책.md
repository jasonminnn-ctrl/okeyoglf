# 09_기능잠금_운영정책

최종 갱신: 2026-03-14

---

## 1. 문서 목적

이 문서는 OkeyGolf OS의 멤버십별 기능 잠금, 기능 노출, 연결 준비 상태, 운영자 override 원칙을 정의하기 위한 기준 문서다.

이 문서의 목적은 다음과 같다.

- trial, standard, pro, enterprise 등 멤버십별 기능 접근 정책을 명확히 한다.
- 준비중 기능을 전부 열어두지 않고, 실제 연결 상태에 따라 활성 / 잠금 / 숨김을 구분한다.
- 고객이 기능 상태를 오해하지 않도록 문구와 상태 표시 원칙을 고정한다.
- 외부 연동 기능이 붙어도 기존 잠금 구조가 흔들리지 않게 한다.
- 운영자 전용 override와 고객용 기능 노출 정책을 분리한다.

---

## 2. 기능 잠금 운영 원칙

기능 잠금은 단순한 임시 장식이 아니라 실제 구독형 SaaS 운영 구조의 일부다.

원칙:
- 준비중 기능을 전부 해제하지 않는다.
- 실제 연결된 기능만 활성화한다.
- 연결 전 단계 기능은 상태를 구분해서 표시한다.
- 잠금과 숨김은 다르게 다룬다.
- 고객이 눌렀을 때 왜 안 되는지 이해할 수 있어야 한다.
- 운영자 override가 있더라도 기본 정책은 멤버십 기준을 따른다.
- 고객용 화면과 운영자용 화면의 통제 수준을 분리한다.

---

## 3. 잠금 상태 정의

### 3-1. enabled
- 의미: 활성
- 설명: 정상 사용 가능
- 기준: 멤버십 조건, 연결 상태, 운영자 설정 조건이 충족된 상태

### 3-2. locked
- 의미: 잠금
- 설명: UI는 보이지만 현재는 사용할 수 없음
- 기준: 멤버십 부족, 권한 부족, 업그레이드 필요, 운영자 설정 미완료 등

### 3-3. hidden
- 의미: 숨김
- 설명: UI에서 완전히 제거
- 기준: 현재 단계에서 고객에게 노출할 필요가 없거나 운영자 전용 기능

## 4. 멤버십 등급 운영 기준

현재 멤버십 등급: `trial`, `standard`, `pro`, `enterprise`

### 4-A. 과금/사용 정책 기본 구조 (확정)

OkeyGolf OS의 과금/사용 정책은 **기간형 멤버십 + 사용량형 크레딧** 이중 구조를 따른다.

#### 기간형 멤버십이 관리하는 범위
- 기능 접근 범위 (enabled / locked / hidden 판정 기준)
- 저장 및 열람 범위
- 기본 스토리지 한도
- 운영 도구 기본 접근
- 결과 복사, 재생성, 내보내기, 공유, 전달 등급별 범위

#### 사용량형 크레딧이 차감되는 범위
차감 대상: AI 생성 (generate), AI 재생성 (regenerate), 시장조사 AI 호출
차감하지 않는 대상: 저장/조회/열람, 관리자 설정 조회, 대시보드 접근, 체크리스트 기본 기능, 결과 복사

---

## 5. 멤버십별 기본 정책

### 5-1. Trial
기본 활성: 대시보드 조회, AI 비서 일부, 체크리스트/요약형 기본 결과, 저장된 결과 기본 저장
기본 잠금: 기본 생성 기능 상당수, 결과 복사/재생성/내보내기/공유, 전담 컨설턴트, 결과 전달

### 5-2. Standard
기본 활성: 기본 생성 기능, 저장/복사/재생성/내보내기/공유, 기본 시장조사
기본 잠금: 전담 컨설턴트 일부, 결과 전달, 고급 시장조사, 외부 연동 자동화

### 5-3. Pro
기본 활성: 전체 생성/저장/복사/재생성/내보내기/공유/전달, 전담 컨설턴트, 고급 시장조사
기본 잠금: 멀티브랜치, 고급 조직 권한

### 5-4. Enterprise
기본 활성: pro 전체 + 멀티브랜치, 고급 권한, 조직 단위 설정

## 7. 운영 리스크 항목 노출 차등 정책

운영 점검 페이지(OpsCheckPage)의 리스크 항목은 아래 3원 구조를 따른다.

### 7-1. AI 제안 리스크 (risk_source: ai_suggested)
- 조직 업종/운영 프로필 기준으로 AI가 놓칠 수 있는 운영 항목/리스크를 제안
- 멤버십 차등: trial은 기본 3~5개 제한, standard 이상 전체 노출 가능
- 현재 상태: 규칙 기반 mock 데이터 (실제 LLM 분석 미연동)

### 7-2. 운영 권장 리스크 (risk_source: ops_recommended)
- 오키골프 운영팀 또는 컨설턴트가 사전 정의한 리스크 템플릿/권장 점검 항목
- 멤버십 차등: trial은 기본 항목만, pro 이상 업종별 심화 항목 노출
- 현재 상태: 고정 템플릿 기반 (운영자 편집 UI 미구현)

### 7-3. 직접 등록 리스크 (risk_source: user_created)
- 사용자가 직접 등록한 운영 항목
- 멤버십 차등: 없음 (전 등급 사용 가능)

### 향후 확장 방향
- 운영자 override로 특정 조직에 AI 제안 리스크 확장/제한 가능
- 컨설턴트 제공 항목은 전담 컨설턴트 요청과 연결하여 프리미엄 서비스로 확장 가능
- 09 문서에 멤버십별 세부 노출 정책이 확정되면 feature-access.ts에 반영

## 8. 외부 연동 상태 표시 원칙

외부연동 상태는 테스트 데이터 존재 여부만으로 `연결됨`으로 표시하지 않는다.

상태 구분:
- **연결 준비** (pending): 구조와 UI는 있으나 실제 외부 호출 없음
- **테스트 연결** (test_connected): 테스트 데이터 삽입, 수동 DB 입력, mock 기반 검수
- **운영 연결** (connected): 실제 API 키, webhook, 운영 매핑, 서버 처리 흐름 검증 완료
- **오류** (error): 실제 연동 시도했으나 실패

원칙:
- 실제 API 키, webhook, 서버 처리 흐름이 검증되기 전에는 `운영 연결`로 표시하지 않는다
- 현재 아임웹 연동은 **테스트 연결** 단계

---

## 9. 운영자 override 원칙

기본 멤버십 정책을 완전히 무시하는 무질서한 예외 처리가 아니라, 제한적 예외 제어 수단이다.

원칙:
- 기본 정책은 membershipCode 기준
- override는 featureKey + membershipCode 기준 분리 구조
- 운영자 전용 페이지에서만 조정
- 고객용 화면에서는 override 구조를 직접 노출하지 않음
- override 사용 시 추적 가능해야 함

### 현재 구현 상태 (2026-03-13)
- `feature_overrides` DB 테이블 존재
- 운영자 `/operator` 페이지에서 override 편집 UI 구현 완료
- org_id / membership_code / feature_key / access_mode / reason 필드 지원

## 10. 업종별 카드 노출 원칙

운영팀과 영업팀의 카드 세트는 선택 업종 기준으로 다르게 노출한다.

원칙:
- 공통 대메뉴는 유지
- 업종에 없는 운영팀 / 영업팀 카드는 기본 노출하지 않음
- 업종 전환 시 카드 세트가 함께 변경

---

## 11. 고객용 문구 원칙

잠금과 연결 준비 상태의 문구:
- 업그레이드 필요
- 운영자 설정 필요
- 연결 준비 중
- 미연결
- 테스트 필요
- 현재 권한으로는 사용할 수 없음

지양:
- 무조건 "준비중"만 반복
- 실제 연결되지 않았는데 동작하는 것처럼 보이는 문구

---

## 12. 고객용 화면과 운영자 화면의 구분 원칙

고객용 화면:
- 실제로 쓰는 기능 중심
- 잠금은 최소한 이해 가능하게 표시

운영자 화면:
- 매핑, 예외 처리, provider 상태, 재처리, override, 내부 정책

---

## 13. 향후 확장 원칙

- 저장공간 quota 실제 enforcement
- 아임웹 주문 기반 자동 멤버십/크레딧 반영
- 알림/발송 실제 provider 연결
- 엔터프라이즈 조직 단위 권한 제어
- 다점포 / 멀티브랜치별 기능 차등

## 14. 현재 단계 요약

완료 또는 확정된 것:
- 멤버십 등급 구분
- 기능 잠금 / 숨김 / 활성 개념 정리
- 결과 관련 잠금 구조
- 운영자 override 구조 및 UI
- 과금/사용 정책 이중 구조 확정
- AI 제안 / 운영 권장 / 직접 등록 3원 리스크 구조
- `/operator` 운영 권장 관리 탭 구현 완료 (업종/조직/사업장 타겟팅)
- 운영 권장 유형: 운영 권장 / 위험신호 / 잘하고 있는 점 / 꼭 확인할 것 / 이번 주 권장 액션

멤버십별 운영 권장 노출 차등 (향후):
- Trial: 기본 운영 권장만 노출
- Standard: AI 제안 + 기본 운영 권장
- Pro: AI 제안 + 전체 운영 권장 + 잘하고 있는 점
- Enterprise: 전체 + 조직별 맞춤 권장 + 위험신호 알림

후속 단계:
- 실제 featureKey별 멤버십 차등 노출 상세 정책 고도화
- AI 제안 리스크의 실제 LLM 연동
- operator_recommendations → 고객 화면 자동 공급 연동
- 저장공간 quota 실제 enforcement
- 엔터프라이즈 권한 확장
