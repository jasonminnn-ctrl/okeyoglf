# 07_시장조사_운영정의

최종 갱신: 2026-03-13

## 1. 문서 목적

이 문서는 OkeyGolf OS의 시장조사 메뉴를 실제 운영 SaaS 기준으로 정의하기 위한 문서다.

---

## 2. 시장조사 운영 원칙

- 시장조사 메뉴는 독립 메뉴로 유지한다.
- 시장조사 결과는 저장된 결과와 연결될 수 있어야 한다.
- 필요 시 전담 컨설턴트 요청으로 전환될 수 있어야 한다.
- AI 마케팅팀 등 다른 메뉴에서 참고할 수는 있으나, 종속시키지 않는다.
- 외부 수집 엔진이 아직 연결되지 않은 상태에서는 AI 내부 분석 기반 결과임을 구분 표시한다.
- 실제 외부 조사 결과, 비교 수치, KPI 해석을 지어내지 않는다.

---

## 3. 현재 구현 상태 (2026-03-13)

### 구현 완료
- `research_requests` DB 테이블 (Supabase, RLS 적용)
- 조사 요청 저장/상태 분기/업종별 템플릿 분기
- 최근 작업 이력 / 저장된 결과 연결
- 결과 삭제 기능
- 저장 결과 DB 연동 (saved_results 연결)
- 결과 내보내기 (TXT 즉시 다운로드)

### 부분 구현 / placeholder
- 결과 생성은 AI 내부 분석 기반 (실제 외부 수집 미연동)
- PDF/DOC/PPT 다운로드: 서버 생성 필요 (미구현)
- 외부 수집 placeholder 구조: `sourceType`, `collectionStatus`, `externalCollectionPlanned` 필드 존재

### 미구현
- 실제 외부 수집 엔진 연동
- 실제 크롤링/API 기반 조사
- soft delete 전환

---

## 4. 시장조사 요청 상태 정의

- `requested`: 조사 요청 시작
- `processing`: 처리 중
- `completed`: 완료
- `consultant_handoff`: 전담 컨설턴트 전환
- `failed`: 실패

---

## 5. 조사형 템플릿 구조

- `competitor`: 경쟁사 조사
- `area`: 지역/상권 조사
- `customer`: 고객/수요 조사
- `promotion`: 프로모션/상품 아이디어 조사
- `operation`: 운영 이슈 조사

---

## 6. 업종별 예시 질문 분기 원칙

업종별로 조사 관점과 예시 질문, 활용 방식이 달라야 한다.
외부 수치나 사실을 새로 생성하지 않는다.
업종별 문맥과 관점만 달라지고, 사실성 원칙은 동일하게 유지한다.

---

## 7. 다른 메뉴와의 연결 원칙

- 시장조사는 독립 메뉴로 유지한다.
- 저장된 결과와 연결될 수 있어야 한다.
- AI 마케팅팀 등은 시장조사 결과를 참고할 수 있다.
- 마케팅팀과의 연결은 약한 연결로 유지한다.
- 시장조사 메뉴를 다른 메뉴의 하위 기능으로 종속시키지 않는다.
