# 01_메뉴구조_정본

## 1. 문서 목적
이 문서는 OkeyGolf OS의 최종 대메뉴 구조와 각 메뉴의 역할을 고정하기 위한 기준 문서다.
향후 러버블 구현, 화면 설계, 저장 구조, 권한 구조, 관리자 설정, 프롬프트 구성은 이 메뉴 구조를 기준으로 확장한다.
메뉴는 임의로 흔들지 않고, 변경이 필요할 경우 반드시 이 문서를 기준으로 검토한다.

## 2. 메뉴 설계 원칙
- 대메뉴는 사용자가 바로 이해할 수 있는 실무형 이름을 사용한다
- 업종별로 메뉴를 바깥에서 나누지 않고, 공통 메뉴 안에서 내부적으로 분기한다
- AI 기능은 각 메뉴 내부에서 작동하며, 사용자에게는 “팀” 또는 “업무” 단위로 보이게 한다
- ROS 엔진명은 사용자 대메뉴로 직접 노출하지 않는다
- AI로 처리 가능한 일과 전담 컨설턴트 요청이 자연스럽게 연결되어야 한다
- 모든 주요 메뉴에는 전담 컨설턴트 요청 CTA를 둘 수 있어야 한다
- 저장된 결과와 관리자 설정은 전체 시스템의 공통 기반 메뉴로 유지한다

## 4. 대메뉴별 정의

### 4-1. 대시보드
역할:
- 서비스 진입 메인 화면
- 현재 상태를 한눈에 확인하는 운영 관제 센터

포함 요소:
- 핵심 KPI
- 최근 생성 결과
- 최근 시장조사
- 최근 전담 컨설턴트 요청
- 이번 주 추천 액션
- 현재 선택 업종
- 멤버십 등급
- 향후 크레딧 요약 영역

### 4-2. AI 비서
역할:
- 사용자가 놓치고 있는 운영 업무를 리마인드하고 다음 액션을 제안하는 실행 비서

포함 요소:
- 오늘의 할 일
- 이번 주 추천 액션
- 놓치고 있는 운영 항목
- 캠페인 추천
- 일정/마감 리마인드
- 업종별 체크리스트
- 빠른 전담 요청 CTA

### 4-3. AI 운영팀

역할:
- 실제 운영 현장에서 자주 쓰는 운영 실무 기능 제공
- 단, 운영팀 내부 카드는 업종에 따라 다르게 분기한다

원칙:
- 대메뉴명은 공통 유지
- 내부 카드는 업종별로 다르게 노출
- 운영팀은 운영 실무 / 배정 / 시설 / 현장 흐름 중심으로 유지
- 가격정책, KPI, 재등록 전환 같은 매출 직접 연관 항목은 업종에 따라 영업팀으로 이동 가능

운영 지원 전담 컨설턴트 CTA


### 4-4. AI 영업팀

역할:
- 고객관리, 전환, 재등록, 상담, 가격정책, 매출과 직접 연결되는 기능 제공
- 단, 영업팀 내부 카드는 업종에 따라 다르게 분기한다

원칙:
- 업종별 고객 / 리드 / 거래처 구조에 맞춰 카드가 달라진다
- 영업팀은 전환, 유지, 가격, 상담, KPI 중심으로 유지한다

영업 지원 전담 컨설턴트 CTA


### 4-5. AI 마케팅팀

역할:
- 캠페인, 카피, 프로모션, 콘텐츠 기획 등 마케팅 실행 지원

포함 요소:
- 마케팅 카피 생성기
- 이벤트 생성기
- 프로모션 기획
- 시즌 캠페인 제안
- 채널 현황
- 콘텐츠 생성기
- 마케팅 지원 전담 컨설턴트 CTA

원칙:
- 시장조사는 독립 대메뉴로 유지한다
- AI 마케팅팀에는 시장조사 메인 카드를 두지 않는다
- 필요 시 결과 화면 또는 하단 CTA에서만 약하게 연결한다



### 4-6. AI 디자인팀
역할:
- 홍보물, 배너, 포스터, 템플릿, 업로드 기반 제작 등 디자인 관련 지원

포함 요소:
- 디자인 요청
- 템플릿 디자인 센터
- 홍보물 문안 + 레이아웃 생성
- 배너/포스터 요청
- 업로드 폼 기반 제작
- 결과물 관리
- 디자인 지원 전담 컨설턴트 CTA

### 4-7. AI 경영지원
역할:
- 문서, 보고, 지시, 회의 정리, 기획안 초안 등 경영지원성 업무 지원

포함 요소:
- 보고서 생성기
- 직원 지시사항 생성기
- 회의 정리
- 운영안 초안
- PPT용 아웃라인
- 승인/검토 메모
- 문서/기획 지원 전담 컨설턴트 CTA

### 4-8. 시장조사
역할:
- 지역, 업종, 키워드 중심의 조사 요청 및 향후 수집/분석 확장 기능

포함 요소:
- 업종
- 지역
- 키워드
- 수집 개수
- 조사 시작 버튼
- 최근 작업
- 결과 영역
- 전담 컨설턴트 연결 CTA

#### 운영 원칙

- 시장조사는 조사 요청 생성, 최근 작업 확인, 저장된 결과 연결까지 포함하는 **독립 워크플로우 메뉴**다.
- 시장조사 결과는 저장된 결과와 연결되며, 필요 시 전담 컨설턴트로 전환된다.
- 외부 수집 엔진 미연동 상태에서는 **AI 내부 분석 기반 결과**임을 구분해서 표시한다.
- 실제 외부 조사 결과, 비교 수치, KPI 해석을 지어내지 않는다.
- 필요 시 "확인 필요", "추가 자료 필요", "현재 입력 기준 가정"으로 표기한다.
- 시장조사는 AI 마케팅팀과 연결될 수 있으나, 마케팅팀에 종속되지 않는다.


### 4-9. 전담 컨설턴트
역할:
- AI로 처리하기 어렵거나 사람이 직접 해야 하는 작업을 요청하는 프리미엄 서비스 메뉴

포함 요소:
- 전담 컨설턴트 요청
- 문서 제작 요청
- PPT 제작 요청
- 운영 분석 요청
- 마케팅 검토 요청
- 디자인 요청

### 4-10. 저장된 결과
역할:
- AI 생성 결과와 전담 요청 결과를 저장하고 다시 열람하는 공통 아카이브

포함 카테고리:
- AI 비서 결과
- AI 운영팀 결과
- AI 영업팀 결과
- AI 마케팅팀 결과

기본 기능:
- 목록 보기

- 카테고리 필터

- 상태값

- 재열람
- 향후 복제/재생성 확장 가능 구조
- 삭제
- 저장량 요약 확인


### 4-11. 관리자 설정
역할:
- 고객사가 자신의 회사 정보, 운영 정보, 내부 사용자 및 기본 사용 환경을 설정하는 고객용 설정 메뉴
- 서비스가 업종별/조직별 컨텍스트를 정확히 이해하도록 사업장 기본값과 내부 운영 기준을 유지하는 입력 화면
- 현재 이용 상태를 고객이 직접 확인할 수 있는 조회 화면을 함께 제공하는 메뉴

포함 요소:
- 회사 기본정보
- 업종 및 운영 형태 설정
- 업종별 운영 기본정보
- 내부 실무 기준정보
- 사용자 관리
- 사용자별 권한 설정
- 기능 및 이용 설정
- 브랜드/자료 설정
- 현재 멤버십 조회
- 업그레이드 안내
- 현재 크레딧 잔액 조회
- 최근 크레딧 사용내역 조회
- 현재 활성 기능/제한 기능 요약 조회


화면 구성 원칙:
- 고객용 관리자 설정은 상단 탭 또는 sticky 서브탭 구조로 정리한다
- 탭은 아래 5개로 사용한다
  - 회사/업종
  - 운영정보
  - 사용자/권한
  - 기능/브랜드
  - 이용현황
- 세로 스크롤이 과도하게 길어지지 않도록 각 탭 안은 카드형 섹션으로 정리한다

권한 분리 원칙:
- 고객용 관리자 설정은 고객사 내부 운영에 필요한 설정과 조회 기능만 다룬다
- 고객용 관리자 설정에서는 프롬프트, 정책, 엔진, 지식, ROS, 멤버십 정책, 크레딧 정책, 기능 노출/비노출 정책을 직접 수정하지 않는다
- 멤버십과 크레딧은 조회 전용으로만 노출하고 정책 편집은 운영자 전용 관리에서만 수행한다

제외 원칙:
- 프롬프트 관리
- 업종별 지침 관리
- 출력 기준 관리
- 금지 규칙 관리
- 지식베이스 관리
- 참고자료/출처 관리
- AI 참조 허용 범위 설정
- 엔진 설정
- ROS 정책
- 멤버십 정책
- 크레딧 정책
- 기능 노출/비노출 설정
- 전담 컨설턴트 전환 규칙
위 항목은 고객용 관리자 설정에 두지 않고 운영자 전용 관리에서만 다룬다

### 5-3. 운영자 외부 연동 탭 (고객 비노출)

- 운영자 전용 `/operator` 내부에 `외부 연동` 탭을 둔다.
- 이 탭은 고객용 관리자 설정 5탭과 분리한다.
- 외부 연동 탭에서는 아래 내용을 다룬다.
  - 연동 상태
  - OpenAI 연동
  - 아임웹 회원/주문 연동
  - 상품 → 멤버십 매핑
  - 포인트상품 → 크레딧 매핑
  - 알림/발송 연동
  - 동기화 이력 / 예외 처리
- 고객용 사이드바 11개 구조는 변경하지 않는다.
- 외부 연동 관련 세부 운영은 운영자 화면에서만 관리한다.

- 영업팀: 리드관리 / 제안관리 / 견적·요금결정 / 미응답·후속관리 / KPI 관리 / 응대 문안

원칙:
- 대메뉴는 공통 유지
- 운영팀과 영업팀만 강하게 업종 분기한다
- 업종에 없는 카드는 노출하지 않는다
- 업종 변경 시 운영팀/영업팀 카드 세트가 함께 바뀌어야 한다

## 8. 메뉴 연결 원칙
- AI 비서는 전체 메뉴를 가로지르는 추천 허브 역할을 한다
- 각 팀 메뉴에서 생성된 결과는 저장된 결과로 연결된다
- 각 팀 메뉴에는 전담 컨설턴트 요청 CTA를 둘 수 있다
- 고객용 관리자 설정은 회사 기본정보와 운영정보를 입력하는 역할을 한다
- 운영자 전용 관리는 모든 모듈의 출력 품질, 정책, 지식 사용 방식, 멤버십, 크레딧, 기능 노출/비노출을 통제한다
- 시장조사는 마케팅팀, 운영팀, 전담 컨설턴트와 모두 연결 가능해야 한다
- 고객용 사이드바에는 기존 11개 대메뉴만 노출한다
- 운영자 관리와 엔터프라이즈는 고객용 대메뉴에 포함하지 않고 내부 role 기반 숨김 route로 운영한다
- 고객용 관리자 설정에서 입력한 회사/운영/브랜드 정보는 중앙 조직 프로필 구조를 통해 AI 생성 컨텍스트에 반영되어야 한다
- 고객용 관리자 설정의 이용현황은 조회 전용으로만 노출하고, 멤버십/크레딧/기능 정책의 직접 편집은 운영자 전용 관리에서만 수행한다


## 9. 구현 시 주의사항
- 메뉴명을 개발 중 임의로 다시 추상화하지 않는다
- 기존 생성기 페이지는 삭제하지 않고 새 메뉴 구조 아래 재배치한다
- ROS 엔진을 사용자 메뉴로 직접 노출하지 않는다
- 상담팀은 독립 대메뉴로 두지 않는다
- 마케팅팀과 디자인팀은 분리 유지한다
- 경영지원실이 아니라 경영지원 명칭을 사용한다
