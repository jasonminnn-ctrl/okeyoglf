# 08_외부연동_정의

최종 갱신: 2026-03-13

---

## 1. 문서 목적

이 문서는 OkeyGolf OS의 외부 연동 구조를 실제 운영 SaaS 기준으로 정의하기 위한 문서다.

---

## 2. 외부 연동 운영 원칙

- API 키는 프론트 코드에 직접 두지 않는다.
- OpenAI 호출은 서버사이드 호출 구조로 처리한다.
- 아임웹은 초기에는 로그인보다 운영 매핑 구조를 우선한다.
- 실제 연결과 placeholder 상태를 명확히 구분한다.
- placeholder를 실제 동작처럼 보이게 만들지 않는다.

---

## 3. 현재 구현 상태 (2026-03-13)

### 구현 완료 (DB 구조 + UI)
- 운영자 `/operator` 외부 연동 탭
- 아임웹 5개 DB 테이블: `imweb_site_connections`, `imweb_product_membership_mappings`, `imweb_point_credit_mappings`, `imweb_order_events`, `imweb_member_links`
- `imweb-repository.ts` 연결
- 외부 연동 탭에서 실데이터 렌더링 검수 완료
- 연동 상태 / OpenAI / 아임웹 회원·주문 / 매핑 / 알림 / 이력 섹션 UI

### 테스트 연결 단계 (실연동 대기)
- 아임웹: DB 구조 존재, 테스트 데이터 기반 확인 가능, 실제 webhook 미수신
- OpenAI: 에지 함수 구조 존재 (`ai-generate`), 실제 API 호출 미연동

### 미구현 (placeholder)
- 아임웹 실제 webhook 수신 및 자동 처리
- 자동 멤버십/크레딧 반영
- 카카오/SMS/이메일 실제 발송

---

## 4. 운영자 외부 연동 탭 구조

외부 연동은 고객용 관리자 설정이 아니라 운영자 전용 `/operator` 내부에서 관리한다.

### 4-1. 연동 상태
상태 구분:
- 연결 준비 (pending)
- 테스트 연결 (test_connected)
- 운영 연결 (connected)
- 오류 (error)

표시 대상: OpenAI, 아임웹, 매핑, 알림/발송

### 4-2. OpenAI 연동
- provider, 모델, API 키 등록 상태, 서버사이드 호출 준비 상태
- 현재 상태: **연결 준비** (에지 함수 구조 존재, 실제 호출 미연동)

### 4-3. 아임웹 회원/주문 연동
- webhook URL 등록 상태, 최근 동기화 상태, 최근 이벤트 수신 여부
- 현재 상태: **테스트 연결** (DB 구조 + 테스트 데이터)

### 4-4. 상품 → 멤버십 매핑
- `imweb_product_membership_mappings` 테이블 기반
- 아임웹 상품 구매 시 어떤 membershipCode를 부여할지 관리

### 4-5. 포인트상품 → 크레딧 매핑
- `imweb_point_credit_mappings` 테이블 기반
- 아임웹 포인트상품 구매 시 내부 creditAmount를 어떻게 반영할지 관리

### 4-6. 알림 / 발송 연동
대상: 카카오 알림톡, SMS, 이메일
현재 상태: **미연결** (provider 구조만 존재)

### 4-7. 동기화 이력 / 예외 처리
- `imweb_order_events` 테이블의 processing_status 기반
- 실패 건, 미매핑 건, 중복 건, 환불 건 추적 구조 존재

---

## 5. 아임웹 연동 우선순위

1. 상품 → membershipCode 매핑 (DB 구조 완료)
2. 포인트상품 → creditAmount 매핑 (DB 구조 완료)
3. 주문 처리 이력 / 예외 처리 (DB 구조 완료)
4. 회원 매칭 (DB 구조 완료)
5. 실제 로그인 / 인증 연계 (미구현, 후순위)

### 아임웹 상품 분류 원칙 (확정)
- **멤버십 상품**: `membershipCode` 매핑 대상. 기간형 구독 등급 변경.
- **포인트상품**: `creditAmount` 매핑 대상. 사용량형 크레딧 충전.

### 아임웹 로그인 연동 검토 원칙
- 현재 Supabase Auth 기반 인증과 아임웹 사이트 로그인은 분리된 세션
- 연동 우선순위는 로그인보다 운영 매핑이 먼저
- 최종 SSO 가능 여부는 공식 지원 범위 확인 전까지 확정하지 않음

---

## 6. OpenAI 서버사이드 호출 원칙

- 프론트엔드는 OpenAI 비밀키를 직접 가지지 않는다
- `supabase/functions/ai-generate/index.ts` 에지 함수 존재
- 실제 호출은 미연동 (mock 응답)
- 향후 Lovable AI 지원 모델 또는 OpenAI API 전환 가능

---

## 7. 기존 구조와의 연결 원칙

- membershipCode는 기존 시스템 값을 그대로 사용한다
- creditBalance, credit ledger 구조와 충돌 나지 않게 외부 연동 필드를 별도 레이어로 둔다
- feature access, override, operator guard를 깨지 않는다
- 준비중 기능을 전부 해제하지 않는다
