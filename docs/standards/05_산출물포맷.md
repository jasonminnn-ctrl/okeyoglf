# 05_산출물포맷

## 1. 문서 목적
이 문서는 OkeyGolf OS에서 생성되는 주요 산출물의 형식과 구조를 정의하기 위한 기준 문서다.
AI가 생성하는 결과물과 전담 컨설턴트가 제공하는 결과물 모두, 저장·재열람·재가공·공유가 가능하도록 일정한 구조를 가져야 한다.

이 문서는 향후 아래 항목의 기준이 된다.
- AI 결과물 출력 형식
- 저장된 결과 상세 구조
- PDF/문서/PPT 출력 기준
- 관리자 설정 내 출력 기준 관리
- 업종별 결과물 템플릿 분기
- 전담 컨설턴트 결과물 업로드 기준

---

## 2. 기본 원칙

### 2-1. 채팅 답변형이 아니라 결과물형으로 만든다
산출물은 대화형 답변이 아니라, 바로 저장하고 재사용할 수 있는 업무 결과물 형식이어야 한다.

### 2-2. 짧은 산출물도 구조를 가진다
문자 문안, 응대 문구, 짧은 실행안이라도 목적과 맥락이 드러나는 구조를 가져야 한다.

### 2-3. 긴 산출물은 문서형 기준을 따른다
보고서, 운영안, 제안서, 회의정리, 분석결과 등은 문서형 구조를 따른다.

### 2-4. 산출물은 저장 가능한 단위로 분리한다
결과물은 하나의 긴 답변 덩어리가 아니라,
- 제목
- 요약
- 본문
- 체크리스트
- 부록
같이 다시 활용 가능한 단위로 분리하는 것을 원칙으로 한다.

### 2-5. 출력 형식은 오키골프 내부 운영 관리에서 제어 가능해야 한다
향후 오키골프 내부 운영 관리에서 결과물의 기본 포맷, 톤, 길이, 표시 항목을 조정할 수 있어야 한다.

## 4. 산출물 유형 분류
OkeyGolf OS의 산출물은 크게 2종으로 나눈다.

### 4-A. 결과형 산출물
문서/보고서/문안/조사 요약/PPT 아웃라인 등 결과물 형태의 산출물.
- 저장된 결과로 보관
- TXT/DOC/PDF/PPT 다운로드 대상

### 4-B. 운영형 산출물
작업, 캠페인, 리마인드, 체크리스트, 실행 항목 등 반복 관리가 필요한 산출물.
- 단순 텍스트 출력으로 끝내지 않고 앱 내부 보드/리스트/체크 구조로 전환 가능해야 한다.
- CSV/XLSX 다운로드 대상 (특히 체크리스트/운영 보드는 XLSX 우선 가치가 높다)
- 결과형 산출물 다운로드와 데이터형/운영형 산출물 다운로드를 혼동하지 말 것
- 완료 가능한 객체는 담당자(assignee_name), 완료자(completed_by_name), 완료시각(completed_at) 메타를 가진다
- 완료 체크 시 인증 사용자 정보를 completed_by 필드에 반영, 해제 시 null 정리

### 4-C. 문맥별 추가 액션
결과 유형에 따라 주요 액션 외에 아래 secondary action을 둘 수 있다:
- 작업으로 저장
- 캠페인으로 저장
- 리마인드로 저장
- 체크리스트로 저장

### 4-D. 세부 유형 분류

## 5. 유형별 포맷 기준

## 5-A. 산출물 접근 / 저장 / 다운로드 원칙

OkeyGolf OS의 산출물은 저장된 결과 메뉴에서만 접근 가능하게 두지 않는다.

기본 원칙은 다음과 같다.

- 결과 생성 직후 사용자는 저장 여부와 관계없이 결과를 바로 볼 수 있어야 한다.
- 결과 생성 직후 사용자는 저장 여부와 관계없이 다운로드 또는 내보내기를 할 수 있어야 한다.
- 저장은 결과를 영구 보관하고 재조회하기 위한 액션이지, 다운로드의 선행 조건이 아니다.
- 저장된 결과와 생성 직후 결과는 가능한 한 동일한 렌더 구조와 동일한 출력 포맷을 사용한다.
- 아직 지원하지 않는 다운로드/내보내기 유형은 비활성 또는 준비중으로 표시하되, 완료처럼 보이게 만들지 않는다.

### 5-A-2. 결과 액션 표준 구조 (Phase 1)

결과 생성 직후 및 저장된 결과 열람 시 노출되는 주요 액션의 기본 순서는 다음과 같다.

1. **저장** — 결과를 영구 보관. 저장은 다른 액션의 필수 전제가 아니다.
2. **내보내기** — TXT 즉시 다운로드를 포함한 다양한 형식 선택. DOC/PDF/PPT는 서버 연결 전까지 비활성(준비중) 표시.
3. **전달** — 이메일 전달, 내부 전달 등. 서버 연결 전까지 준비중 표시.
4. **다시 생성** — 동일 입력 기반 재생성. 크레딧 차감 정책 적용.
5. **전담 컨설턴트 요청** — AI 결과를 전담 컨설턴트에게 전달하는 액션.

제외 원칙:
- `TXT 다운로드` 단독 버튼은 두지 않는다. TXT는 내보내기 다이얼로그 안에서 선택한다.
- `공유`는 실제 외부 공유 연동 전까지 기본 주요 액션으로 노출하지 않는다.
- `복사`는 필수 주요 액션이 아니라 섹션별 보조 유틸리티로 제공한다.

## 5-B. 채팅형 결과 렌더 원칙

AI 작업실(보조 AI 작업실 포함)에서 생성된 결과는 단순 말풍선 텍스트로만 끝내지 않는다.

원칙:
- 결과는 채팅 안에서도 결과 카드 형태로 렌더한다.
- 결과 카드의 주요 액션은 5-A-2의 표준 순서(저장 → 내보내기 → 전달 → 다시 생성 → 전담 컨설턴트 요청)를 그대로 따른다.
- 문맥에 맞는 경우, 주요 액션 아래에 운영 객체 전환 보조 액션(작업으로 저장 / 캠페인으로 저장 / 리마인드로 저장 / 체크리스트로 저장)을 추가로 노출한다.
- 복사는 섹션별 보조 유틸리티로 제공하며, 주요 액션에 포함하지 않는다.
- 저장 여부와 관계없이 결과 생성 직후 바로 내보내기가 가능하다.
- 저장은 영구 보관 목적이며, 채팅형 결과 접근의 필수 조건이 아니다.

## 7. 업종별 산출물 차이
같은 산출물 유형이라도 업종에 따라 구성과 강조 포인트가 달라진다.

### 골프연습장
- 운영 실행안
- 재등록 액션
- 타석/레슨 관련 체크리스트
- 프로모션 실행 문안

### 골프장
- 잔여타임 운영안
- 가격/채널 운영 보고서
- 팀수 확보 제안
- 패키지 프로모션 기획안

### 골프아카데미
- 수강생 유지 전략안
- 강사 운영 체크리스트
- 교육 상품 구조안
- 학부모/회원 안내 문안

### 골프샵
- 판매 제안형
- 프로모션 기획안
- 상품 홍보 문안
- 매장 운영 보고서

### 피팅샵
- 상담 스크립트형
- 예약/노쇼 관리 실행안
- 후기 유도 문안
- 브랜드별 제안형 결과물

### 골프회사
- 영업 제안형
- 파트너 운영 보고서
- 브랜드/상품 성과 문서
- B2B용 제안 자료

## 8. 저장된 결과와의 연결 원칙
모든 주요 산출물은 저장된 결과에 들어갈 수 있어야 한다.

저장 시 필요한 기본 요소:
- 제목
- 모듈
- 업종
- 결과 유형
- 생성일
- 작성자
- 요약
- 상세 본문
- 메모
- 상태값

향후 추가 가능 요소:
- 버전
- 첨부자료
- 다운로드 파일
- 공유 이력
- 재생성 이력

## 9. 다운로드/공유 확장 방향
향후 아래 확장을 고려한다.
- PDF 다운로드
- 문서형 다운로드
- PPT 아웃라인 내보내기
- 카카오 공유용 요약형
- 내부 링크 공유
- 전담 컨설턴트 결과물 전달 이력

현재 단계에서는 실제 구현이 아니라 구조와 포맷 기준만 먼저 확정한다.

## 10. 오키골프 내부 운영 관리에서 제어해야 할 항목
향후 오키골프 내부 운영 관리에서 아래를 제어할 수 있어야 한다.

- 기본 출력 유형
- 모듈별 기본 포맷
- 업종별 출력 템플릿
- 길이/요약 수준
- 제목 규칙
- 금지 표현
- 확인 필요 문구 자동 삽입 여부
- 저장 시 기본 카테고리
- 전담 컨설턴트 전환 기준

## 11. 전담 컨설턴트 결과물 기준
전담 컨설턴트가 직접 처리한 결과물도 이 문서의 구조 기준을 최대한 따른다.

원칙:
- 결과물은 고객이 다시 열람 가능한 구조로 정리
- 제목과 요약이 있어야 함
- 전달용 파일이 있더라도 본문 요약을 함께 남김
- 버전 구분 가능해야 함
- 고객이 이해할 수 있는 문장으로 정리

---

## 12. 구현 시 주의사항
- 산출물은 채팅용 답변처럼 흘러가면 안 된다
- 문서형, 실행형, 저장형 구조를 우선한다
- 짧은 산출물도 목적과 맥락을 드러내야 한다
- 업종에 맞지 않는 결과물 형식을 억지로 쓰지 않는다
- 전담 컨설턴트 요청으로 넘어간 결과물도 동일 기준으로 정리한다
