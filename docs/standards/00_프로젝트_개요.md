## 2. 프로젝트 한 줄 정의
골프업 전체를 대상으로, 운영에 필요한 업무를 AI와 전담 컨설턴트 요청 구조로 지원하는 구독형 운영 SaaS

## 2-1. 프로젝트 주체 및 서비스 정체성

### 프로젝트 주체
이 프로젝트의 주체는 오키골프이며, 골프업 실무 경험을 바탕으로 운영 SaaS를 설계하고 구축하는 것을 목표로 한다.

### 오키골프가 지향하는 방향
오키골프는 단순한 마케팅 대행이나 문구 생성 서비스가 아니라,
골프업 운영 전반에 필요한 업무를 AI와 전담 컨설턴트 구조로 지원하는 운영 시스템을 지향한다.

즉, 사용자가 운영에 필요한 판단, 실행안, 보고, 지시, 기획, 디자인 요청, 시장조사, 저장/재활용까지 하나의 SaaS 안에서 해결할 수 있도록 돕는 것을 목표로 한다.

### 이 서비스가 해결하려는 문제
골프업 현장에서는 여전히 엑셀, 문자, 감(感), 개별 담당자 역량에 의존하는 경우가 많다.
이 프로젝트는 그런 운영 방식을 줄이고, 반복되는 판단과 실행 업무를 구조화하여,
누구나 더 일관되고 빠르게 운영할 수 있도록 돕는 것을 목표로 한다.

### 서비스의 본질
이 서비스는 단순 AI 챗봇이 아니다.
골프업 운영에 필요한 실무를 지원하는 운영 SaaS이며,
AI가 처리 가능한 영역은 AI가 처리하고, 사람이 직접 개입해야 할 영역은 전담 컨설턴트 요청으로 연결하는 구조를 가진다.

### 제품의 핵심 흐름 (2026-03-13 기준)
1. AI가 결과를 생성한다 (문서형 산출물)
2. SaaS가 그 결과를 운영 객체(작업/캠페인/리마인드/체크리스트)로 관리한다 (운영형 산출물)
3. 사용자는 직접 입력/수정하거나 AI 비서에게 자연어로 요청할 수 있다
4. 모든 운영 객체는 담당자 배정, 완료 추적, CSV/XLSX 내보내기가 가능하다
5. 전담 컨설턴트는 AI 한계를 넘는 작업을 사람 중심으로 처리한다

### 맞춤형 응답을 위한 전제
이 프로젝트 안에서는 아래 관점을 항상 반영해야 한다.
- 추상적인 아이디어보다 실제 현장에서 바로 쓸 수 있는 실행안이 중요하다
- 할루시네이션을 줄이기 위해 프롬프트, 지식베이스, 출력 기준을 통제해야 한다
- 업종별 언어와 KPI, 실무 흐름을 구분해야 한다
- AI가 확신할 수 없는 내용은 확인 필요로 표시해야 한다
- 결과물은 저장, 재열람, 재활용 가능한 구조여야 한다

### 프로젝트 주체의 작업 성향
이 프로젝트는 아래 작업 성향을 중요하게 반영한다.
- 실무에 바로 적용 가능한 결과를 선호한다
- 구조와 기준을 먼저 잡고 실행하는 방식을 선호한다
- 숫자, KPI, 실행안, 체크리스트를 중시한다
- 감성적 문구보다 운영에 도움이 되는 결과물을 선호한다
- 결과물은 저장/재사용/전달 가능한 형태여야 한다
- 불확실한 내용은 확실한 것처럼 포장하지 않는다


## 3. 프로젝트 목적
이 프로젝트의 목적은 골프업 종사자들이 실제 운영에 필요한 기능을 하나의 SaaS 안에서 사용할 수 있도록 구조를 설계하고 구현하는 것이다.
단순한 AI 챗봇이나 문구 생성기가 아니라, 실제 운영 실무에 쓰이는 시스템을 목표로 한다.

## 4. 핵심 방향
- 공통 대메뉴를 사용하고 내부에서 업종별로 분기한다
- AI가 처리할 수 있는 일은 AI가 처리한다
- AI로 처리하기 어려운 일은 전담 컨설턴트 요청으로 넘긴다
- 모든 결과물은 저장되고 재열람, 재활용될 수 있어야 한다
- 고객사는 프롬프트를 직접 설정하는 구조가 아니라, 회사 기본정보와 업종별 운영 기준값을 입력하는 구조를 사용한다
- 오키골프 내부 운영자는 프롬프트, 지식베이스, 출력 기준, 금지 규칙, AI 참조 정책, 기능 정책을 통제한다
- 시스템은 고객 입력값을 직접 프롬프트 편집값이 아니라 운영 컨텍스트로 사용한다
- 할루시네이션을 줄이기 위해 내부 지식베이스, 출력 기준, 프롬프트 정책, 검토 규칙을 중요하게 설계한다
- 러버블로 실제 구현 가능한 구조를 우선으로 한다
- AI가 답변하거나 생성하는 기능과, 사용자가 직접 입력/수정/필터링/관리하는 운영형 데이터 화면을 함께 설계한다
- KPI, 고객관리, 이용권, 상담, 요청처리, 크레딧 이력 같은 운영 데이터는 표형/리스트형 관리 구조로도 다룰 수 있어야 한다
- 결과물 다운로드는 PDF/문서/PPT뿐 아니라 CSV/XLSX 같은 데이터형 내보내기까지 지원한다
- 운영형 페이지(체크리스트/운영점검/캠페인/리마인드)는 직접 입력 + AI 비서 + 추적 + export 하이브리드 구조를 따른다

## 8. 각 메뉴의 역할 요약

### 대시보드
핵심 KPI, 최근 결과, 최근 시장조사, 최근 전담 요청, 추천 액션을 한눈에 보는 메인 화면

### AI 비서
오늘 할 일, 놓치고 있는 운영 항목, 일정/마감, 주간 추천 액션, 업종별 체크리스트를 제안하는 실행 비서 기능.
운영형 카드(운영 점검/캠페인/리마인드/체크리스트)는 DB 기반 운영 객체 관리 페이지로 직접 진입한다.

### AI 운영팀
운영 진단, 요금결정, 잔여타임, 타임관리, 타석관리, 레슨관리, KPI 분석 등 운영 실무 중심 기능

### AI 영업팀
고객관리, 재등록 관리, 미방문 관리, 판매 제안, VIP 관리, 패키지 제안, 응대 문안 등 매출/고객 전환 중심 기능

### AI 마케팅팀
카피 생성, 이벤트 기획, 프로모션 기획, 채널 운영안, 시즌 캠페인, 시장조사 연계 기능

### AI 디자인팀
디자인 요청, 템플릿 디자인 센터, 홍보물 문안 및 레이아웃 생성, 배너/포스터 요청, 업로드 폼 기반 제작 기능

### AI 경영지원
보고서 생성, 직원 지시사항 생성, 회의 정리, 운영안 초안, PPT 아웃라인 등 문서/기획 지원 기능

### 시장조사
업종, 지역, 키워드 기준의 조사 요청 및 향후 데이터 수집/분석 확장 기능

### 전담 컨설턴트
AI로 처리하기 어려운 문서, PPT, 운영 분석, 마케팅 검토, 디자인 요청 등을 오키골프가 직접 처리하는 프리미엄 요청 영역

### 저장된 결과
AI 생성 결과, 시장조사 결과, 전담 요청 결과를 카테고리별로 저장하고 다시 열람하는 아카이브

### 관리자 설정
조직 기본정보, 업종 및 운영 형태, 업종별 운영 기본정보, 사용자/권한, 기능 및 이용 설정, 브랜드/자료 설정 등 고객사가 자신의 운영 컨텍스트를 입력하는 설정 영역


## 9. 설정 구조의 중요성

이 프로젝트에서 설정 구조는 단순 환경설정이 아니라 서비스 운영 품질과 고객사별 컨텍스트를 나누어 관리하는 핵심 영역이다.

고객사가 직접 프롬프트나 금지 규칙을 편집하는 구조는 지양한다.

대신 아래 두 층으로 분리한다.

### 9-1. 고객용 관리자 설정
고객용 관리자 설정은 아래 항목을 중심으로 구성한다.
- 조직 기본정보
- 업종 및 운영 형태 설정
- 업종별 운영 기본정보
- 내부 실무 기준정보
- 사용자/권한 설정
- 기능 및 이용 설정
- 브랜드/자료 설정

### 9-2. 오키골프 내부 운영 통제
오키골프 내부 운영 통제 영역은 아래 항목을 관리한다.
- 프롬프트 운영
- 업종별 지침 운영
- 출력 기준 운영
- 금지 규칙 운영
- 지식베이스 운영
- 참고자료/출처 운영
- AI 참조 허용 범위 운영
- 프롬프트 버전 및 활성 정책 운영
- ROS 엔진 및 정책 운영
- 전담 컨설턴트 전환 규칙 운영
- 기능 정책 및 크레딧 정책 운영

고객이 입력한 회사 기본정보와 업종별 운영 기준값은 AI가 직접 참고하는 사업장 컨텍스트로 사용하고,
AI 정책 자체는 오키골프 내부 운영자가 통제하는 구조를 유지한다.


## 10. 제품 구현 원칙
- 러버블로 실제 구현 가능한 흐름으로 설계한다
- 처음부터 모든 외부 연동을 한 번에 붙이지 않는다
- 구조 → 세부 메뉴 → 저장/첨부 → AI 연결 → 크레딧 → 다운로드/공유 → 외부 연동 순서로 진행한다
- 기존 대시보드 톤과 품질은 최대한 유지한다
- 기존에 만든 생성기 페이지는 삭제하지 않고 새 구조 아래 하위 도구로 재배치한다
- 사용자 메뉴는 이해하기 쉬운 실무형 이름을 사용한다
- ROS 엔진명은 사용자 대메뉴가 아니라 내부 로직/설정 구조로 사용한다

## 11. 현재 구현 상태 (2026-03-13 기준)

### 구현 완료
- Supabase Auth 기반 인증 (로그인/로그아웃/비밀번호 재설정)

- 결과 액션바 표준화 (저장→내보내기→전달→다시생성→컨설턴트)
